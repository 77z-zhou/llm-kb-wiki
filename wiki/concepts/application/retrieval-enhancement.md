# RAG 检索增强技术

## 一、增强检索器

### 混合搜索 (Hybrid Search)
- **稀疏检索 (BM25)**：关键词精确匹配，适合术语/ID/缩写
- **密集检索 (向量搜索)**：语义模糊匹配，适合口语化/长尾查询
- **结合方式**：分数加权融合 → 兼顾精确和语义

### 重排序 (Re-ranking) — 两阶段检索
1. **召回**：快速方法召回大候选集 (Top 50)
2. **精排**：Cross-Encoder 对候选集精细打分 → Top 5
- Cross-Encoder 直接比较查询和文档文本，精度远超向量相似度

## 二、优化查询

| 技术 | 原理 | 适用场景 |
|:-----|:-----|:---------|
| **Multi-Query** | LLM 生成多个不同角度查询，合并结果 | 查询表述模糊 |
| **HyDE** | LLM 生成假设性答案，用答案嵌入去检索 | 答案与文档文本形式更相似 |
| **Sub-Querying** | 复杂问题分解为子问题，分别检索 | 多跳推理问题 |

## 三、优化索引结构

### 小块引用大块 (Small-to-Large)
- 检索时匹配小块（高精度）
- 送给 LLM 的是对应的大块（丰富上下文）

### 图索引 (Graph Indexing)
- LLM 提取实体和关系 → 构建知识图谱
- 图查询 + 向量检索互补
- 适合多跳推理

## Lost in the Middle 问题

LLM 倾向忽略长上下文**中间部分**的信息。

**缓解方法**：
1. **文档重排序**：最相关的放开头和结尾
2. **减少检索数量**：严格控制 Top-K
3. **指令化提示**：明确要求关注所有上下文
4. **微调 LLM**：训练其利用中间信息

## 补充见解

- **重排序是性价比最高的优化手段**：只需加一个 Cross-Encoder，就能显著提升检索质量
- **混合搜索已成为 RAG 系统标配**：纯向量搜索在精确匹配场景表现不佳
- **Query 改写是容易被忽视但效果显著的环节**：用户原始查询往往不是最佳的检索 query
