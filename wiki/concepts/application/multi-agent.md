# 多智能体系统 (Multi-Agent System)

## 核心定义

多个自主的 LLM Agent 在同一环境中协同工作，通过通信、协作、竞争或协商解决单个 Agent 难以处理的复杂问题。

## 相比单 Agent 的优势

| 优势 | 说明 |
|:-----|:-----|
| **分工专业化** | 每个 Agent 专注特定角色（如产品经理/程序员/测试） |
| **并行处理** | 子任务分配给不同 Agent 同时执行 |
| **鲁棒性** | 单个 Agent 失败不影响整体 |
| **视角多样性** | 辩论/协商方式避免单一思维局限 |

## 引入的新复杂性

1. **通信协议**：Agent 间如何有效沟通？需要标准化消息格式
2. **协调机制**：谁来领导？如何分配任务？如何解决冲突？
3. **社会行为**：信任、欺骗、联盟等复杂社会现象
4. **状态一致性**：多个 Agent 对共享环境的认知同步
5. **信用分配**：任务成功/失败时如何评估每个 Agent 的贡献

## A2A (Agent-to-Agent) 框架

**普通 Agent 框架 vs A2A 框架的核心区别**：

| 维度 | 普通 Agent 框架 | A2A 框架 |
|:-----|:---------------|:---------|
| 关注点 | 个体的内部实现 | 群体的交互标准 |
| 抽象层次 | 应用层 | 协议层 |
| 类比 | 如何造一辆车 | 如何建交通规则 |

## 补充见解

- **AutoGen (微软) 是目前最流行的多 Agent 框架**，支持角色扮演和群组对话
- **多 Agent 的成本是单 Agent 的倍数**，需要在效果和成本间权衡
- **实际应用中"伪多Agent"更常见**：用同一个 LLM 扮演不同角色，而非真正独立的多个模型实例
