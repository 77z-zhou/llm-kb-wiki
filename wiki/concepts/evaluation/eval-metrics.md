# LLM 评估指标与基准

## 传统指标的局限性 (BLEU/ROUGE)

| 问题 | 说明 |
|:-----|:-----|
| 无语义理解 | 只做表层词汇匹配 |
| 无法评估事实性 | 无法检测幻觉 |
| 忽略多样性 | 惩罚与参考答案不同但优秀的回答 |
| 长文本差 | 无法评估连贯性和逻辑性 |
| 无视推理过程 | 只比较最终答案字符串 |

## 综合性基准测试

| 基准 | 侧重点 | 形式 | 评估目的 |
|:-----|:-------|:-----|:---------|
| **MMLU** | 知识广度与学科解题 | 57科目四选一 | 跨学科知识储备 |
| **Big-Bench** | 能力边界与未来潜力 | 200+多样化任务 | 通用智能与涌现能力 |
| **HumanEval** | 代码生成 | 164编程题+单元测试 | 编写正确代码能力 |
| **GSM8K** | 数学推理 | 小学应用题 | 多步思维链推理 |
| **ARC** | 科学推理 | 挑战性选择题 | 科学常识与推理 |
| **HellaSwag** | 常识推理 | 句子续写选择 | 常识判断 |

### HumanEval 评估方法
- **pass@k 指标**：生成 k 个代码样本，至少一个通过全部单元测试即算通过

## 补充见解

- **MMLU 正在被更强的基准替代**：如 MMLU-Pro、GPQA (研究生级别)
- **评估基准本身也会"过拟合"**：模型在训练数据中见过测试题会导致虚高分数
- **真实场景评估比基准测试更重要**：基准高分不等于产品体验好
