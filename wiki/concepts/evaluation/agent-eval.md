# Agent 评估

## 为什么比 LLM 评估更难？

| 维度 | 基础 LLM | Agent |
|:-----|:---------|:------|
| 本质 | 静态文本生成器 | 动态决策者 |
| 交互 | 单轮输入→输出 | 多轮有状态交互 |
| 环境 | 确定、可复现 | 动态、不可预测 |
| 答案 | 有参考标准 | 开放式，无唯一正确答案 |

## 评估维度对比

| 维度 | LLM | Agent |
|:-----|:----|:------|
| 核心 | 单个回答质量 | 整个任务完成过程 |
| 主要指标 | 准确性/流畅性/相关性 | 任务成功率/效率/鲁棒性/自主性 |
| 过程指标 | 无 | 成本/延迟/步骤数/纠错能力 |
| 评估环境 | 静态数据集 | 交互式环境 |

## Agent 基准测试

| 基准 | 领域 | 特点 |
|:-----|:-----|:-----|
| **WebArena** | 网页浏览操作 | 高度逼真的网站模拟 |
| **AgentBench** | 通用综合评估 | 8个不同环境(OS/DB/KG/游戏) |
| **GAIA** | 复杂任务 | 需组合多种工具，对人简单对AI难 |

### 基准构建方法
1. **环境**：Docker 容器沙箱，本地托管网站副本，Mock API
2. **任务**：高层次目标（非步骤指令）+ 可程序化验证的成功标准
3. **评估**：自动检查环境最终状态是否满足条件

## 关键过程指标

- **效率**：Token 消耗、API 费用、总耗时、步骤数
- **鲁棒性**：错误处理能力、抗干扰能力
- **自主性**：需人类干预次数、行为可解释性、计划遵循度

## 补充见解

- **Agent 评估的最大挑战是可复现性**：动态环境导致两次运行结果可能完全不同
- **过程指标比结果指标更有价值**：一个 3 步完成的 Agent 比 20 步完成但结果相同的 Agent 质量更高
- **线上 A/B 测试 + 用户反馈是最终的评估标准**：实验室基准无法完全替代真实场景
