# LLM-as-a-Judge 与专项评估

## LLM-as-a-Judge

用强大 LLM（如 GPT-4o）作为裁判评估另一个 LLM 的输出质量。

### 工作流程
评估 Prompt（问题+回答+评估准则）→ 裁判 LLM → 结构化评分+理由

### 优点
1. **可扩展**：大规模自动化评估，加速迭代
2. **一致性**：固定准则下评估标准统一
3. **可定制**：灵活设计评估维度

### 潜在偏见

| 偏见类型 | 表现 |
|:---------|:-----|
| **位置偏见** | 偏爱第一个呈现的答案 |
| **冗长偏见** | 给更长回答更高分 |
| **自我偏好** | 偏爱与自己风格相似的回答 |
| **知识有限** | 可能识别不出专业领域错误 |
| **过于宽容** | 对有害内容判断比人类宽松 |

## 专项能力评估设计

### 事实性/幻觉
- 构建知识库 QA 数据集 + 对抗性问题
- 方法：精确匹配 / LLM-as-a-Judge / FaithScore

### 推理能力
- 使用 GSM8K / LogiQA / Big-Bench Hard
- 方法：结果评估（最终答案）+ 过程评估（推理步骤）

### 安全性
- 使用 AdvBench / SafetyBench 对抗性提示
- 指标：**拒绝率**（成功拒绝有害问题）+ **误伤率**（错误拒绝正常问题）

## 补充见解

- **LLM-as-a-Judge 与人类评估的相关性约 80%**：可用于快速迭代，但关键决策仍需人工
- **缓解偏见的技巧**：随机化 A/B 顺序、控制答案长度、多次评估取平均
- **专项评估的关键在于数据集质量**：对抗性样本越多样化，评估越可靠
